## Summary

InternSwipe v1.0 delivers a complete backend API for a Tinder-style internship discovery and application tracker. Students can sign up, browse eligible internships, swipe to express interest, and submit applications — all through a secured REST API with session-based authentication, input validation, and rate limiting.

---

## Features delivered

### Authentication
- Email/password sign-up, login, and logout via Supabase Auth.
- Session cookies are automatically refreshed on every request via Next.js middleware.
- Three public endpoints: `POST /api/auth/signup`, `POST /api/auth/login`, `POST /api/auth/logout`.

### User profiles
- Create and update user profiles with name, phone, links, and preference data.
- `GET /api/profile` and `PUT /api/profile` with Zod validation on all fields.

### Job discovery
- `GET /api/jobs` with full pagination support (page, limit).
- Filter by eligibility status (`ELIGIBLE` / `NOT_ELIGIBLE`), company name, or full-text search across title, company, summary, and location.
- Sort by `created_at`, `company`, or `title` in ascending or descending order.
- 25 curated internship listings seeded (15 eligible, 10 not eligible).

### Swipe actions
- `POST /api/swipe` records LEFT or RIGHT swipe actions per user per job.
- Validates that the job exists before recording.

### Application pipeline
- `POST /api/apply` validates resume ownership, job eligibility, and duplicate prevention.
- Creates `Application` and `SubmissionLog` records for every attempt (including failures).
- Ineligible jobs are logged as FAILED applications with error details.
- Duplicate applications return 409 Conflict.

### Application history
- `GET /api/applications` returns all user applications with related job details and submission logs, ordered by most recent first.

### Resume signed URLs
- `GET /api/resume/signed-url` generates 15-minute expiry download URLs via Supabase Storage.
- Validates that the requesting user owns the resume before generating the URL.

### Security baseline
- **Auth guards:** Centralized `requireAuth()` helper applied to all 6 protected route handlers. Returns 401 for unauthenticated requests.
- **Rate limiting:** In-memory sliding-window rate limiter on `POST /api/apply` — 10 requests per minute per user. Returns 429 with `Retry-After` header.
- **Signed URLs:** Resume files are only accessible through time-limited signed URLs generated by an authenticated endpoint.
- **Secret management:** All credentials stored in `.env.local` (gitignored). Example file contains placeholder values only.

### Database
- 7 tables: `users`, `profiles`, `resumes`, `jobs`, `swipe_actions`, `applications`, `submission_logs`.
- Prisma ORM with PrismaPg adapter for Supabase connection pooling.
- Version-controlled migrations.
- Seed script with 25 internship listings.

### Testing
- Vitest unit tests for `RateLimiter` class (sliding window, expiry, multi-user isolation).
- Unit tests for `requireAuth()` helper (authenticated and unauthenticated flows).

### Documentation

## Known limitations

1. **No frontend UI.** The API layer is complete but the swipe deck, profile forms, history view, and auth pages need to be built as React components.
2. **No resume upload endpoint.** The signed URL download endpoint exists, but the file upload flow (accept PDF, validate size, upload to Supabase Storage, create Resume record) is not implemented.
3. **In-memory rate limiter.** Resets on server restart. Acceptable for a class project; would need Redis or database-backed storage for production.
4. **No Playwright E2E tests.** Only unit tests exist. End-to-end browser testing was not completed.
5. **No Vercel deployment.** The application runs locally only. Production deployment configuration is documented but not executed.
6. **No CI merge blocking.** GitHub Actions pipeline runs lint and typecheck but branch protection rules are not configured.
7. **Eligibility flags not audited.** Seeded job eligibility flags have not been verified against original postings.

## Next steps

If development continues beyond v1.0, the following are recommended priorities:

1. **Build the frontend UI** — swipe deck with card animations, login/signup forms, profile page, and application history view.
2. **Implement resume upload** — `POST /api/resume/upload` with PDF-only validation, 5 MB size limit, and Supabase Storage integration.
3. **Add Playwright smoke tests** — cover signup, login, swipe, apply, and history flows.
4. **Deploy to Vercel** — connect the GitHub repository and configure environment variables.
5. **Configure branch protection** — require CI checks to pass and at least one review before merging to main.
6. **Replace in-memory rate limiter** — use Redis or a database counter for persistence across server restarts and multiple instances.
7. **Add job ingestion API** — automated job scraping or API integration to keep listings fresh.
